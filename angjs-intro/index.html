Background and overview

AngularJS1

  • Open-source JavaScript framework, maintained by Google
  • JavaScript MVC Framework for building Single-Page Applications (SPAs)
  • Notable emphasis on testability and embeddability
  • MIT licensed

Similar Libraries and Frameworks

  • Backbone.js
    Minimalistic, low-level, make-your-own-stack library
  • Ember.js
    Minimal boilerplate, opinionated, "developer-friendly", very Angularish
  • Knockout
    MVVM framework. Declarative bindings, automatic UI refresh
  • Polymer
    Component-based application library (pre-alpha)

Conceptual Overview

The Zen of Angular2

"Angular is built around the belief that declarative code is better than imperative when it comes to building UIs and wiring software components together, while imperative code is excellent for expressing business logic."

Design goals2

  • Decouple DOM manipulation from app logic (→ testability)
  • Regard application testing as equal in importance to application writing (testing difficult affects code structuring)
  • Decouple server side and client side (→ reusability, testability)
  • Guide developers (UI design → business logic → testing)
