Tools and Design

Tools

browserify

  • require for browser
  • parses requires at compile time and creates a combined javascript file
  • should work on ie7+
  • enables to use transformations such as CoffeeScript and Babel
  • allows to embed non-javascript files directly using plugins such as stringify and browserify-plain-jade
  • node-type requires
    • // math.js
module.exports = {
  sum: function(a, b) {
    return a + b
  }
}

    math = require('./math')

math.sum(1, 2)

npm

  • node package manager
  • contains libraries for both front and back such as jQuery, lodash and Angular
  • browserify automatically looks for installed npm packages when requiring
npm install lodash --save-dev

_ = require('underscore')

Build tools

  • gulp – a build system that allows to run browserify and other tasks
  • browsersync – enables live reloading and synchronization between devices. Can be integrated into gulp.
  • sass, stylus, etc – preprocessors for css. Allow using a more concise syntax and add new features
  • jade – preprocessor for html. Allows writing html in a tree-like fashion

EcmaScript 6

  • ES6 is the next version of JavaScript
  • partial support in modern browsers, Babel allows to use it now (ie9+)
  • notable features
    • arrow function
    • destructuring
    • let, const
  • more https://github.com/lukehoban/es6features

ES6 – let, const

  • block scoping instead of function scoping
  • let is like var, const is for constants
  • Strong mode proposal forbids var so it might be a good idea to start using let instead.
function() {
  let x = 3
  if (true) {
    let x = 4
    console.log(x) // 4
  }
  console.log(x) // 3
}

for (let i = 0; i < 10; i++) {
    console.log(i);
}
console.log(i); // ReferenceError: i is not defined

ES6 – Arrow functions

  • shorter function definition, always anonymous
  • lexical this, the function retains its parent's scope. Compare to let.
values.map(v => v + 1)

(u, v) => u * v

x => {
  console.log(x)
  return 3 * x
}

y => ({
  value: y,
  other: 3
})

() => console.log('bar')

ES6 – native modules

  • works with browserify and babel
// math.js
export function sum(x, y) {
  return x + y
}

console.log(sum(2,3))

Other tools

  • bower – Frontend package manager. No good way to fix version numbers, seems to break arbitrarily at some point. Use npm instead.
  • webpack – combines require, transformations and build into one tool. A notable alternative to browserify and gulp.
  • jspm, system.js – alternative package manager and require tool. See a nice demo.
  • grunt – an older alternative to gulp. Nothing wrong in itself but very verbose configuration file. Use something else for new projects.

Angular – Composition through directives

  • Directives can be used to define custom html tags
  • Directives may have their own controller
  • So why not use them to create custom html components
  • This provides full encapsulation, modularity and reusable components
  • Full example with gulp and browserify
.directive('participants', () => ({
    restrict: 'E',
    scope: {},
    template:
      '

    controller: ['$scope', '$filter', 'participantService', ($scope, $filter, participantService) => {

        $scope.participants = $filter('orderBy')(participants, 'name')
      })
    }]

Angular minification

  • Angular's dependency injection works on parameter names
  • Minification renames parameters -> DI does not work
  • Solution: Angular allows you to pass an array together with the function ['$scope', function ($scope) {}]
  • Angular will inject the listed objects in the order they are gicing regardless of the actual function parameter names
function test1 () {
  return {
    controller: ['$scope', '$filter', function ($scope, $filter) {
      $scope.test = $filter
    }]
  }
}
function test1(){return{controller:["$scope","$filter",function(t,e){t.test=e}]}}

Best practices

  • Use browserify, webpack, etc for modularity
  • Use directives for composition
  • Use nested directives
  • Most (if not all) of your controllers should be part of a directive
  • Move most of your business logic to services
  • If your controller is longer than 100 lines, you are likely doing something wrong
