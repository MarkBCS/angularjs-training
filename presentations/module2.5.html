Composition through directives

  • Directives can be used to define custom html tags
  • Directives may have their own controller
  • So why not use them to create custom html components
.directive('participants', () => ({
    restrict: 'E',
    template:
      '
Event Participant list

       
Search participants: 

       
  1. {{ participant.name }}
',
    controller: ['$scope', '$filter', 'participantService', ($scope, $filter, participantService) => {
      participantService.participants.success((participants) => {
        $scope.participants = $filter('orderBy')(participants, 'name')
      })
    }]
}));

Summary

Main Building Blocks in Angular Apps

  • Controllers
  • Filters
  • Directives
  • Expressions
  • Services and Factories
  • Constants and Values
  • Modules

Dependencies between these building blocks are managed with Angular's dependency injection mechanisms (Providers)

When Should I Create ...

  • Filter: when you find yourself repeating same data manipulation pattern
  • Directive: when you detect a common usage pattern in your templates
  • Service: when you detect a common usage patterns inside controllers or want to share functionality between controllers
  • Constant or Value: when you find yourself repeating same constant or value in your components
  • Module: to capsulate independent part of your application
